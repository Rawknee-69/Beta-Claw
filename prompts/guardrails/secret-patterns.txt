# Secret and API key patterns for output scanning
# Format: pattern_name | regex_pattern | description
openai_api_key | sk-[A-Za-z0-9]{20,} | OpenAI API key
anthropic_api_key | sk-ant-[A-Za-z0-9\-]{20,} | Anthropic API key
google_api_key | AIza[A-Za-z0-9\-_]{35} | Google API key
aws_access_key | AKIA[A-Z0-9]{16} | AWS access key ID
aws_secret_key | [A-Za-z0-9/+=]{40} | AWS secret access key (contextual)
github_token | ghp_[A-Za-z0-9]{36} | GitHub personal access token
github_oauth | gho_[A-Za-z0-9]{36} | GitHub OAuth token
github_fine_grained | github_pat_[A-Za-z0-9_]{22,} | GitHub fine-grained PAT
stripe_secret | sk_live_[A-Za-z0-9]{24,} | Stripe live secret key
stripe_test | sk_test_[A-Za-z0-9]{24,} | Stripe test secret key
bearer_token | Bearer\s+[A-Za-z0-9\-._~+/]+=* | Bearer authentication token
basic_auth | Basic\s+[A-Za-z0-9+/]{20,}={0,2} | Basic authentication header
jwt_token | eyJ[A-Za-z0-9\-_]+\.eyJ[A-Za-z0-9\-_]+\.[A-Za-z0-9\-_]+ | JSON Web Token
slack_token | xox[baprs]-[A-Za-z0-9\-]{10,} | Slack bot/app/user token
slack_webhook | hooks\.slack\.com/services/T[A-Z0-9]+/B[A-Z0-9]+/[A-Za-z0-9]+ | Slack webhook URL
twilio_api_key | SK[a-f0-9]{32} | Twilio API key
sendgrid_api_key | SG\.[A-Za-z0-9\-_]{22}\.[A-Za-z0-9\-_]{43} | SendGrid API key
mailgun_api_key | key-[A-Za-z0-9]{32} | Mailgun API key
heroku_api_key | [0-9a-fA-F]{8}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{4}-[0-9a-fA-F]{12} | Heroku API key (UUID)
private_key_pem | -----BEGIN (RSA |EC |DSA )?PRIVATE KEY----- | PEM private key header
database_url | (postgres|mysql|mongodb(\+srv)?):\/\/[^\s]+ | Database connection string
redis_url | redis:\/\/[^\s]+ | Redis connection string
generic_secret | (secret|password|passwd|token|api_key|apikey)\s*[:=]\s*['"][^\s'"]{8,} | Generic secret assignment
npm_token | npm_[A-Za-z0-9]{36} | NPM access token
pypi_token | pypi-[A-Za-z0-9\-]{50,} | PyPI API token
